# プレゼンテーションスライドの普遍的デザイン原則

スライド制作時に必ず参照する。全レイアウト・全スタイルに共通で適用。
ブランド固有のカラー・フォント・サイズは `shared/slide-design.md` を参照。
日本語の改行・短文化は `references/slide-copy-fitting.md` を参照。

---

## 1. レイアウトと視覚的構成

| 原則 | ガイドライン |
|------|-----------|
| **CRAP原則** | Contrast(対比), Repetition(反復), Alignment(整列), Proximity(近接)。全要素をグリッドに沿って整列、関連項目をグループ化、重要情報は色・サイズ・配置の対比で際立たせる |
| **視覚的階層** | Z/Fパターンで視線を制御。タイトル=最大、ポイント=中、詳細=小。情報の重要度に応じた階層を構築 |
| **余白** | スライド面積の30%以上を余白として確保。情報の境界を明確にし、視覚的圧迫感を軽減 |
| **三分割法** | 重要な画像・アイコン・キーメッセージは三分割線の交点付近に配置。中央配置より魅力的な構図になる |

---

## 2. タイポグラフィと可読性

| 原則 | ガイドライン |
|------|-----------|
| **フォントサイズ** | タイトル36pt以上、本文24pt以上（最小18pt）。会場最後列からの視認性を確保 |
| **フォント制限** | 1プレゼン最大2書体。見出し=サンセリフ体、本文=可読性の高いゴシック体 |
| **行間・字間** | 行間1.2〜1.5倍。箇条書きの間隔は通常行間より広く |
| **テキスト最小化** | 1行20文字以内、箇条書き4つまで。キーワードや短いフレーズで構成 |
| **改行禁則** | 助詞・句読点・単位を行頭に出さない。見出しは2行まで、1文字だけの末尾行を作らない |

---

## 3. 配色とアクセシビリティ

| 原則 | ガイドライン |
|------|-----------|
| **3色ルール** | ベース(70%): 背景、メイン(25%): 文字・図形、アクセント(5%): 強調・データポイント |
| **コントラスト比** | 文字と背景のコントラスト比4.5:1以上（WCAG AA準拠）必須 |
| **意味の重複** | 色だけで情報を伝えない。太字・アンダーライン・アイコン・パターンを併用 |

---

## 4. コンテンツ戦略と構造

| 原則 | ガイドライン |
|------|-----------|
| **ワンスライド・ワンメッセージ** | 1枚=1つの明確な結論(Takeaway)。余計な情報は削除 or 別スライドに分割 |
| **アサーション・エビデンス** | 上部に主張(完全な文章)、下部に証拠(図解・データ)を配置 |

## 4.5. ビジュアルファースト原則（優先順位の明文化）

良いスライドは「文字の流し込み」ではなく「ビジュアルコミュニケーション」で設計する。
**テキスト箇条書きはビジュアル化できなかった情報にのみ使う。**

### 設計の思考順序（必ず守る）

各スライドを設計するとき、以下の順で考える:

1. **このメッセージを図解・グラフ・表・写真で伝えられないか？** → 伝えられるなら、それを主役にする
2. 図解を選んだら `slide-diagram-catalog.html`（D01〜D30）から最適なパターンをコピーする
3. テキストは「主役を補足する最小限」に絞る。箇条書きは原則3項目以内

### デッキ全体のビジュアル比率（目標）

| スライド種別 | 目標比率 |
|---|---|
| 図解・フロー・マトリクス・ベン図が主役 | 30%以上 |
| グラフ・チャート・数値が主役 | 15%以上 |
| 写真・キービジュアルが主役 | 10%以上 |
| テキスト主体（箇条書き・ステートメント） | 45%以下 |

### ビジュアル変換ルール

| メッセージの性質 | 使うビジュアル |
|---|---|
| 手順・プロセスがある | フロー図（横・縦）、ステップ番号付き |
| 要素の関係・構造を見せたい | マトリクス、ベン図、階層図、ハブ＆スポーク |
| 数値の変化・比較を見せたい | 棒グラフ、折れ線グラフ、円グラフ |
| Before/After・対比がある | 2カラム比較、スプリットスライド |
| 感情・リアリティを高めたい | 写真、人物ビジュアル |
| 重要な一言を伝えたい | 大きなテキスト＋余白（ステートメント型） |

---

## 5. ビジュアルとデータ可視化

| 原則 | ガイドライン |
|------|-----------|
| **高品質な画像** | ぼやけた画像・透かし入り画像は使用禁止。裁ち落としで視覚的インパクトを最大化 |
| **図解の簡素化** | 3ステップ以内のフロー図、中心から広がるマインドマップ形式など、シンプルに |
| **データの強調** | グラフ背景をグレーにして目立たなくし、注目データポイントのみアクセントカラーで着色 |

---

## AI実装時の適用ルール

1. 全スライドで余白30%以上を確保（`padding: 4cqw 7cqw`を基本）
2. フォントサイズの階層は3段階（slide-h2 / slide-body / slide-caption）を基本
3. 色は`--c-base` / `--c-main` / `--c-accent`の3色システムを厳守
4. 1スライドの箇条書きは最大4項目
5. コントラスト比4.5:1をCSS変数設定時に検証
6. 強調は色+太字の併用。色だけに頼らない
7. **フチ文字（`-webkit-text-stroke`）禁止**
8. **ドロップシャドウ（`box-shadow`）原則禁止**。区切りはborderで代替
9. **光彩・グロー（`text-shadow`のぼかし / `filter: drop-shadow()`）禁止**
10. **過剰な装飾禁止**（グラデーションボーダー・波形背景・浮き出し風装飾等。スタイルファイルで明示許可されたもの以外使用不可）
