# 新案件セットアップ

新しい案件が来たときに呼び出してください。フォルダ作成からCLAUDE.md準備まで一緒にやります。

---

## ステップ1：基本情報をヒアリング

以下の項目を順番に聞いてください。

```
【新案件セットアップ】

以下を教えてください：

1. 案件名（フォルダ名に使います）
   例：oyama-jinja / komatsu-sake / crater-lp など

2. クライアント名（正式名称）

3. 案件の種類は何ですか？
   例：WebサイトLP / コーポレートサイト / ロゴ / グラフィック など

4. 案件ごとに管理しますか？クライアントごとに管理しますか？
   - 案件ごと：[案件名]/ でフォルダを作る
   - クライアントごと：[クライアント名]/[案件名]/ でフォルダを作る
     ※クライアントごとの場合、既存フォルダがあればその中に作ります
```

---

## ステップ2：フォルダ構成を確認・作成

### 案件ごとの場合
```
/Projects/claude/[案件名]/
  └ CLAUDE.md
  └ brief/
  └ output/
```

### クライアントごとの場合
まず `/Projects/claude/` 内に同名フォルダがないか確認する。
- あれば → その中に案件フォルダを作る
- なければ → クライアントフォルダごと作る

```
/Projects/claude/[クライアント名]/
  └ [案件名]/
      └ CLAUDE.md
      └ brief/
      └ output/
```

フォルダ構成を提示して確認を取ってから作成する。

---

## ステップ3：CLAUDE.mdをヒアリングして作成

以下を聞いてCLAUDE.mdに書き込む。

```
追加で教えてください：

5. 担当者名・連絡先（わかれば）

6. 納期（わかれば）

7. トンマナ・世界観（わかれば）
   例：高級感・モダン / 親しみやすい・ポップ / シンプル・清潔感 など

8. カラーイメージ（わかれば）
   例：ネイビー・ゴールド / ホワイト・グリーン など

9. 参考サイト・参考資料（URLや資料名）

10. その他の要望・禁止事項（わかれば）
```

---

## CLAUDE.mdのテンプレート

```markdown
# [案件名] 案件ルール

## クライアント情報
- 会社名：
- 担当者：
- 案件種類：
- 納期：

## トンマナ・世界観

## カラーイメージ

## 参考資料
- brief/ フォルダを参照

## 禁止事項・注意事項

## 進行メモ
```

## ステップ4：GitHubに登録する

フォルダ作成後、以下のコマンドを実行してGitHubに登録する。
これにより次回からファイルを編集・pushするだけで自動でNetlifyに反映される。

```bash
git add [フォルダ名]/output/[ファイル名.html]
git commit -m "新案件追加：[案件名]"
git push
```

- **HTMLファイルのみ指定する**（`git add フォルダ/` と書かない）
- node_modules・.env・画像など不要なファイルが上がるのを防ぐため
- 初回のみ必要（2回目以降は同じコマンドでpushするだけ）

push完了後、以下をユーザーに伝える：
```
GitHubに登録しました。
公開URLはこのルールで作れます：

https://crater-to.netlify.app/[フォルダ名]/output/[ファイル名]

例：output/sitemap.html を作ったら
→ https://crater-to.netlify.app/[フォルダ名]/output/sitemap.html
```

## ステップ5：資料の確認

```
資料はありますか？

あれば brief/ フォルダに入れてください。
Claudeに「この資料を読んで」と言えば内容を反映します。

準備できたら次のフェーズへ進みましょう。
何から始めますか？
- /ヒアリング（要件まとめ）
- /web-ia（コンテンツマップ）
- その他
```

---

## ルール

- フォルダ作成前に必ず構成を確認する
- 不明な項目は「わかれば」で聞く（なくてもOK）
- 資料はClaudeが読んで初めて使える（置くだけでは読まない）
- クライアントごとの場合は既存フォルダを必ず確認する
- フォルダ作成後は必ずGitHubに登録する（ステップ4）
