# web-wireframe スキル

## 目的
コンテンツマップで確定した「何を」を「どう配置するか」に変換するフェーズ。
実際のLP設計で使えるレベルのワイヤーフレーム（配置図）を作成。

---

## プロセス（4段階）

### Phase 1: 要件確認
- コンテンツマップが確定済みか確認
- セクション構成、セクション内の要素が確認済み
- テキストコンテンツが手元にあるか確認

### Phase 2: HTML ワイヤーフレーム作成
- 各セクションの **配置（レイアウト）** を決定
  - 推奨パターン：**左テキスト、右ビジュアル** の2カラムレイアウト
  - セクションによってはカード配置、グリッド配置を使用
  - 視覚的な優先度をどう表現するか
- テキストコンテンツを全て埋め込む
- モノクロ＆シンプル
  - 背景色：白/グレー程度
  - テキスト色：黒
  - 罫線：グレー
- **アイコン・画像プレースホルダー**
  - **重要：SVG要素として直接HTMLに埋め込む**（CSS生成の::beforeは避ける）
  - Figmaに送信時に正しく認識されるため
  - データURL（SVGコードを直接記述）を使用
  - 例：`<svg xmlns="..." viewBox="..." width="40" height="40"><path .../></svg>`
- **セクション番号の配置**（レビュー用）
  - ページに `padding-left: 70px;` を設定（セクション番号用スペース）
  - セクション番号を `position: absolute; left: -70px; top: 80px;` で配置
  - スクロール時にコンテンツとともに移動するため `position: absolute` 使用
  - 番号は灰色の円形背景（幅32px、高さ32px、border-radius: 50%）
- ヘッダー・フッター含める
  - 全幅表示対応（負のマージンと幅調整で実現）
- プレビューで確認（各セクションのバランスが取れているか）

### 推奨レイアウトパターン

**Pattern 1: 左テキスト、右ビジュアル（推奨）**
```
┌─────────────────────────────────────┐
│                                     │
│  テキスト領域    │  ビジュアル領域   │
│  ・見出し        │  （グレーボックス）│
│  ・説明文        │                  │
│  ・リスト        │                  │
│  ・CTA ボタン    │                  │
│                  │                  │
└─────────────────────────────────────┘
```
→ 使用セクション：KV、サービス概要など

**Pattern 2: カード配置（複数要素）**
```
┌──────┬──────┬──────┐
│ Card │ Card │ Card │
├──────┼──────┼──────┤
│ Card │ Card │      │
└──────┴──────┴──────┘
```
→ 使用セクション：課題紹介、メリット など

**Pattern 3: 比較表**
```
┌─────┬─────┬─────┐
│  A  │  B  │  C  │
├─────┼─────┼─────┤
│ ... │ ... │ ... │
└─────┴─────┴─────┘
```
→ 使用セクション：サービスコンセプト など

**Pattern 4: タイムライン（縦）**
```
●─ Period 1: Content
●─ Period 2: Content
●─ Period 3: Content
```
→ 使用セクション：成果イメージ など
### Phase 3: やり取りで調整
ユーザーから以下の指摘を受け、HTML で修正
- 「このセクションの配置順序を変えたい」
- 「ここのカード数を3個から4個に」
- 「左右の配置を逆にしたい」
など

小さな調整なら HTML で即修正。大型変更は Figma へ移行の判断。

### Phase 3.5: CSS実装のポイント

**ページレイアウト**
```css
.page {
    background: white;
    padding-left: 70px;  /* セクション番号用スペース */
}
```

**セクション番号**
```css
.section-number {
    position: absolute;
    left: -70px;  /* 左側のスペースに配置 */
    top: 80px;    /* セクションのパディングと同じ値 */
    width: 32px;
    height: 32px;
    border-radius: 50%;
    background: #e0e0e0;
    display: flex;
    align-items: center;
    justify-content: center;
}

.section {
    padding: 80px 40px;  /* top値と合わせる */
    position: relative;   /* 番号の親要素として機能 */
}
```

**ヘッダー・フッター全幅対応**
```css
.header {
    margin-left: -70px;           /* パディング分を戻す */
    padding-left: calc(40px + 70px);  /* パディング + 左スペース */
    width: calc(100% + 70px);     /* 幅を調整 */
}

.footer {
    margin-left: -70px;
    padding-left: 110px;          /* 左スペース + コンテンツパディング */
    width: calc(100% + 70px);
}
```

**SVGアイコン（Figma対応）**
```html
<!-- CSS生成ではなく、SVG要素として直接埋め込む -->
<div class="icon">
    <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 40 40" width="40" height="40">
        <path d="..." stroke="#666666" stroke-width="1.5" />
    </svg>
</div>
```

### Phase 4: 確定
- レイアウトが確定
- 次フェーズ「web-layout-frame」へ進む

## 出力物

### HTML ワイヤーフレーム
```
wireframe-full.html
├── ヘッダー（ロゴ・グローバルナビ）
├── セクション1-9（各セクション）
│   ├── セクションナンバー
│   ├── 見出し
│   ├── コンテンツ配置（実際のレイアウト）
│   ├── テキスト（全て埋め込み済み）
│   └── 画像・アイコン（SVG要素として埋め込み）
├── フッター
└── レスポンシブ対応
```

### 備考
- モノクロ・シンプル
- 実装時に参考になるレベルの配置
- 色・フォント・装飾はまだ決めない（次フェーズ）

### Figma送信対応
- **必須：アイコン・画像はSVG要素として直接埋め込む**
  - CSS生成（::before、background-image）は避ける
  - Figmaのキャプチャで正しく認識されないため
- HTMLの head に Figma キャプチャスクリプトを追加：
  ```html
  <script src="https://mcp.figma.com/mcp/html-to-design/capture.js" async></script>
  ```
- ワイヤーフレーム完成後、generate_figma_design ツールで Figma に送信
- キャプチャIDを使用してローカルURLを開き、ポーリングで完了確認

## よくある修正パターン

### セクション番号が中央に寄ってしまう場合
**原因**：セクションコンテナが `display: flex` でセクション番号がフレックスアイテムになっている
**解決**：セクション番号を `position: absolute` にして、flex の影響を受けないようにする

### セクション番号がスクロール時に固定される場合
**原因**：`position: fixed` を使用している
**解決**：`position: absolute` に変更し、親要素のセクションが `position: relative` になっていることを確認

### セクション番号の位置がセクションごとに異なる場合
**原因**：各セクションのパディング（padding-top）が異なっている
**解決**：全セクションのパディングを統一する（例：80px）
```css
.section, .section.merits, .section.timeline {
    padding: 80px 40px;  /* 全て同じ値に統一 */
}
```

### ヘッダー・フッターが左側でカットされている場合
**原因**：ページレベルの左パディングを無視している
**解決**：ヘッダー・フッターに負のマージンと幅調整を追加
```css
.header, .footer {
    margin-left: -70px;        /* パディング分を相殺 */
    width: calc(100% + 70px);  /* 右側も合わせる */
}
```

## 次フェーズへのバトンタッチ

ワイヤーフレームが確定したら、`web-layout-frame` スキルへ：
- レイアウトは確定 → レイアウト案は変更しない
- ここから色・フォント・デザイン要素を追加
- 最終的な見た目（デザインレイアウト）を決定

---

## 必須機能（全ワイヤーフレームに必ず入れる）

### 1. コメント機能
`wf-bar-right` に「💬 コメント追加」ボタンを入れる。押下後、ページ上の任意の位置をクリックしてピンを立て、コメントを入力できる。文字修正・要望もピンで入力してもらう運用。

### 2. 使い方ポップアップ
`wf-bar-right` の左端に「？ 使い方」ボタンを設置。クリックで以下4項目のポップアップを表示：

1. 一番上のメニューバーはワイヤー確認用のため、実際のデザインには含まれない
2. 各セクションの番号はコンテンツの識別用。最終的なデザインには入らない
3. 右上「💬 コメント追加」を押してから対象箇所をクリック → ピンを置き内容を入力。文字修正もここに記入
4. デザインはまだ未着手。現在はレイアウト・構成の確認段階

ポップアップは暗い背景（#1a1a1a）のモーダル形式。外側クリックで閉じる。

### 3. パスワード保護
pushするHTMLにのみ入れる。ローカル確認用には不要。パスワード：`view`

### 実装の順番（wf-bar-rightの中）
```html
<div class="wf-bar-right">
  <button id="wf-help-btn">？ 使い方</button>   ← 使い方（左）
  <span id="wf-count">0件</span>
  <button id="wf-toggle-resolved">解決済みを隠す</button>
  <button id="wf-comment-btn">💬 コメント追加</button>  ← コメント（右）
</div>
```

## 使用シーン

```
コンテンツマップ確定
        ↓
web-wireframe (← いまここ)
    ↓
HTML ワイヤーフレーム作成
    ↓
ユーザーからレイアウト修正の指摘
    ↓
HTML で調整（小型）or Figma へ（大型）
    ↓
レイアウト確定
    ↓
web-layout-frame へ移行
        ↓
デザイン参考分析 ← 別スキル
    ↓
色・フォント・装飾を決定
    ↓
最終レイアウト + デザイン確定
        ↓
コーディング開始
```
