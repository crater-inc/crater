# スキル定義：AI社員採用

## スキル名
**AI社員採用** - 新しいAI社員（スキル）のヒアリング・設計・実装

## 目的
「こんな業務を担当するAI社員が欲しい」という要望を受けて、
ヒアリング → 設計確認 → SKILL.md 実装まで一気通貫で行う。
クレーター・APOLLOS 両社のAI社員を採用できる。

---

## 起動時の動作

引数があれば案件フォルダの `CLAUDE.md` を読み込む。
なければ即ヒアリングフェーズへ進む。

```
例：/AI社員採用
→ ヒアリング開始

例：/AI社員採用 02_APOLLOS
→ 02_APOLLOS/CLAUDE.md を読んでからヒアリング開始
```

---

## Phase 1：ヒアリング（5問）

起動したら以下の5問を**まとめて**提示する。
全部まとめて聞く（1問ずつ聞かない）。

```
【AI社員採用ヒアリングシートです。以下の5問に答えてください。】

1. どんな業務を担当させたい？（具体的な作業内容）
2. クレーター / APOLLOS / 両社どちら向け？
3. インプットは何？（資料・URL・テキスト・画像・指示文など）
4. アウトプットは何？（HTML・ドキュメント・提案書・分析レポートなど）
5. 使うタイミングは？（案件開始時 / 日常的 / 定期的 など）
```

---

## Phase 2：設計確認

ヒアリング回答をもとに以下を設計してユーザーに提示し、GoサインをもらってからPhase 3へ進む。

### 設計書フォーマット

```
【AI社員設計書】

■ スキル名：〇〇担当（日本語）
■ 対象：クレーター / APOLLOS / 両社
■ 一言説明：〜するスキル
■ トリガーワード：「〇〇して」「〇〇担当」など
■ インプット：〜
■ アウトプット：〜
■ 処理フロー：
  1. 〇〇を確認する
  2. 〇〇を分析する
  3. 〇〇を生成する
  4. 確認・修正ループ
■ 禁止事項：〜

このまま実装しますか？変更があれば教えてください。
```

## 設計原則

### スキルを設計するときの心がけ
- **1スキル1フェーズ**：複数フェーズをまとめない（確認のタイミングがずれる）
- **インプット明確化**：スキルが受け取る情報を明確にする
- **アウトプット明確化**：何が出てくるかを明確にする
- **ユーザー確認ポイントを設ける**：作りすぎない、確認してから進む
- **禁止事項を書く**：やりがちなNG行動を明示する

### CRATER / APOLLOS の違い

| | CRATER | APOLLOS |
|---|---|---|
| 事業 | ブランディング・デザイン制作 | 認知特化・価値が届く状態をつくる |
| 主なアウトプット | Web・グラフィック・ブランドガイド | LP・SNS・コンテンツ・認知設計 |
| スキルの傾向 | 制作フロー系（IA・WF・コーディング）| マーケ系（SNS・LP・分析・コピー）|
