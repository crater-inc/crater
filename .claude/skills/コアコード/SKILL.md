# スキル定義：コアコード

## このスキルが目指すこと

ブランドの「最初のイメージ」になる記号を決める。

コアコードは、ブランド側が「これが強みです」と主張したいものではなく、**ユーザーが見たときに「ちょっと気になる」「欲しいかも」「バズりそう」と感じる要素**のこと。

素材・色・価格帯・形・場所・売り方・味・体験——どんな要素でもよい。大事なのは「ブランドにしかないか」より先に「**ユーザーの感情が動くか**」。感情が動いて、かつそのブランドにしかないもの、がコアコード。

---

## 起動時の動作

```
/コアコード [ブランド情報 or 「ヒアリングから」]

例：/コアコード 加賀野菜の金時草を使ったサプリメントブランド
例：/コアコード → ヒアリングから始める
```

---

## フロー

### STEP 1：ブランド情報のインプット

以下を確認する。すでに情報があれば省略してよい。

| 項目 | 確認内容 |
|------|---------|
| 商品・サービス | 何を売っているか。形・味・成分・価格帯など |
| 見た目・体験 | 色・パッケージ・形状・使うシーン |
| 産地・背景 | 原材料・場所・人物・ストーリー |
| ターゲット | 誰が買うか。どんな気持ちのときに買うか |
| 競合 | 似ているブランドは何か |
| 「売れそう」な直感 | ブランドオーナーが「ここ刺さりそう」と思っているポイント |

ヒアリングは一問一答ではなく、**「売れそう・刺さりそう」な直感から深掘り**して進める。

---

### STEP 2：コアコード候補を洗い出す

ブランドの要素を「ユーザー目線」で広く洗い出す。

洗い出す軸（このブランドに関係する軸だけ使えばよい）：

| 軸 | 例 |
|----|---|
| 見た目・色 | パッケージの金色、スティック1本の細さ |
| 価格帯・価値感 | 1包300円という「ちょっと贅沢」なライン |
| 形・形態 | 溶かして飲む、毎朝の儀式になる形 |
| 素材・産地 | 日本にしかない野菜、地域ブランドの力 |
| 味・体験 | 青臭くない、毎日続けられる味 |
| 売り方・場所 | EC限定、贈り物にもなる |
| ストーリー | 誰が作っているか、なぜ作ったか |
| ターゲット文脈 | 「治療ではなく、輝くための準備」という文脈 |
| コミュニケーション | 言えないから伝える、イメージで売るブランド |

候補を3〜5個並べて、ユーザーへの「刺さり度」と「そのブランドにしかない度」の両面で評価する。

---

### STEP 3：コアコードを1つに絞る

選定の基準：

1. **ユーザーが「おっ」となるか** → ワクワク・欲しい・バズりそう・見てみたいの感情が動くか
2. **競合が同じことを言えないか** → 完全には真似できない角度か
3. **ブランド全体を引っ張れるか** → 名前・デザイン・コピーの方向性がここから出てくるか

迷ったら「ユーザー感情を動かすか」を優先する。独自性より先に「刺さるか」。

コアコードの言語化：
1. **日本語タイトル**（20字以内。断言する。ブランド側の言葉ではなくユーザーが感じる言葉で）
2. **英語サブタイトル**（格調ある英語。固有名詞・素材名・場所名があるとよい）
3. **説明文**（150〜250字。「なぜこれがコアコードか」をブランド側の視点で語る。ただしユーザーへの刺さり方にも触れる）

---

### STEP 4：ブランドコード一覧を定義する

コアコードを支える独自資産・特徴を **3〜5個** 定義する。これが「その他のブランドコード」になる。

コアコードほど尖っていなくてよい。ブランドの世界観を支える要素を整理する感覚で。

各コードの構成：
1. **カテゴリラベル**（産地・素材 / 製品形態 / コミュニケーション / ターゲット文脈 / 価格帯・価値 など）
2. **タイトル**（20字以内）
3. **説明文**（60〜100字）

---

### STEP 5：HTMLセクション出力

**必ずHTMLファイルとして保存する。**

```
output/core-code.html
```

すでにブランドボードHTMLがある場合は、そのデザインシステム（カラー変数・フォント）に合わせて出力する。

#### HTML構成（参考テンプレート）

```html
<!-- Core Code セクション -->
<div class="card" style="padding:0;overflow:hidden;">

  <!-- コアコード（メイン） -->
  <div style="display:grid;grid-template-columns:1fr 1fr;gap:60px;padding:52px 56px;align-items:center;">
    <div>
      <div style="/* Core Code ラベル（小さく・英字） */">Core Code　|　コアコード</div>
      <div style="/* 日本語タイトル（大きく・力強く） */"> ← STEP 3で定義 </div>
      <div style="/* 英語サブタイトル（ゴールドカラー） */"> ← STEP 3で定義 </div>
    </div>
    <div>
      <!-- 説明文 ← STEP 3で定義 -->
    </div>
  </div>

  <!-- ブランドコード一覧（アコーディオン） -->
  <details class="accordion">
    <summary>Other Codes　|　その他のブランドコード</summary>
    <div class="accordion-body">
      <!-- STEP 4で定義した3〜5個のコード -->
      <!-- 01 / 02 / 03 ... 番号付きカード形式 -->
      <!-- 末尾に「追加予定」の破線カードを入れる -->
    </div>
  </details>

</div>
```

スタンドアロンHTMLとして出力する場合は以下を含める：
- Google Fonts（EB Garamond + Noto Sans JP）
- CSSカラー変数（ブランドカラーに応じて設定）
- アコーディオンの開閉JS（CSS transitionで滑らかに）
- スマホ対応（`@media (max-width: 768px)` でグリッドを1カラムに）

## アウトプットのクオリティ基準

- コアコードを読んだユーザーが「おっ、ちょっと気になる」と思えるか
- 「ブランド側が言いたいだけ」になっていないか
- 説明文が抽象的・マーケ用語だらけになっていないか
- 英語サブタイトルが「本物感」「高級感」「固有感」を持っているか

---

## 禁止事項

- 情報が不十分でも止まらない → HTMLファイルを必ず出力する
- コアコードを2つ以上にしない（サブコードに入れる）
- 競合も同じことを言えるような当たり障りのないコアコードを選ばない
- ユーザー感情より「ブランドの主張」を優先しない
- 「〇〇を大切にしています」「品質にこだわっています」などの凡庸な言葉を使わない
