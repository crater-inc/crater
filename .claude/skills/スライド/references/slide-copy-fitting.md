# Slide Copy Fitting

日本語スライドの見出し・本文を、自然な改行位置で収めるための基準。
HTMLモード、構成モードのどちらでも使う。

## When To Read

- レイアウトを選んだ後、見出しや本文を入れる前
- `\n` や `<br>` を手で入れる前
- overflow 修正時に、フォント縮小以外の打ち手を判断したい時

## Core Rule

- 改行は最後に入れる
- まず改行なしで起草し、次に短文化し、最後に必要最小限の手動改行を入れる
- 収まりが悪い時は、**フォント縮小より先に** コピー短文化、レイアウト変更、幅調整を試す

## Line Budgets

| 文脈 | 目安文字数/行 | 最大行数 | 超えた時の優先アクション |
|---|---:|---:|---|
| 表紙・中表紙 (`slide-hero`) | 8〜12 | 2 | コピー短文化、主語削除、対比語だけ残す |
| 強い見出し (`slide-h1`) | 12〜18 | 2 | 主張を1つに絞る、補足を本文へ移す |
| タイトル (`slide-h2`) | 14〜20 | 2 | 小見出し化、カラム再配分 |
| 小見出し (`slide-h3`, `slide-title`) | 16〜22 | 2 | 語尾を削る、並列語を減らす |
| 本文 (`slide-body`) | 18〜24 | 3 | 箇条書き化、文を2つに分ける |
| 注記 (`slide-caption`, `slide-micro`) | 12〜20 | 2 | 出典・条件だけ残す |
| 左右分割の強調文 | 8〜10 | 2 | body ブロックへ逃がす |
| 数値の単位ラベル | 4〜8 | 1 | `nowrap` 化する |

## Workflow

1. まず改行なしで書く
2. その要素の文字数予算と行数上限を決める
3. 予算を超えるなら、修飾語・接続語・重複語を削る
4. それでも長ければ、本文へ移すか別レイアウトに変える
5. 最後に文節の切れ目で `\n` または `<br>` を入れる
6. 行頭助詞、1文字だけの末尾行、単位だけの独立行がないか確認する

## Forbidden Breaks

- 行頭が助詞: `は / が / を / に / で / と / の / へ / や / も`
- 行頭が句読点や閉じ括弧: `、 。 ） 」 』 】`
- 行末が開き括弧や接続だけで終わる: `（ 「 『 【 ・`
- 数値と単位が分離する: `12` と `%`、`3` と `倍`
- 末尾行が1文字だけになる

## Rewrite Patterns

- 悪い: `現場でAIを / 使いこなすには`
- 良い: `現場でAIを使いこなす / には何が要るか`

- 悪い: `売上を3倍に / する方法`
- 良い: `売上を3倍にする / 方法`

- 悪い: `導入で / 95%改善`
- 良い: `導入で95% / 改善`

## Layout Decisions Before Font Reduction

- 見出しが3行になるなら `center` から `left-right` へ移す
- 本文が4行を超えるなら 箇条書き化する
- 強調文が10文字を超えるなら `text-accent` ではなく `slide-body` に落とす
- 数値と説明が干渉するなら `split-2` や `metric` 系レイアウトを使う

## Delivery Check

- visible text に `LAYOUT`, `PHOTO`, `CATEGORY`, `SECTION 01`, `Lorem ipsum` が残っていない
- 主要見出しは声に出して1息で読める
- 日本語の助詞が行頭に出ていない
- 本文の最終行が極端に短くない
- 自動検査後に主要スライドを目視しても違和感がない
